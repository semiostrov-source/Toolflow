## Requests feature

The requests feature will cover service, write-off, and related operational flows that act on inventory and warehouses. It will eventually host UI components, domain types, and utilities for creating, reviewing, and tracking different request types, while keeping the request lifecycle and validation rules scoped to this module.

As the product evolves, this module will support richer request workflows, status transitions, and cross-linking with inventory and warehouse data, while maintaining a clear boundary so that request-specific logic remains cohesive and easy to reason about.
