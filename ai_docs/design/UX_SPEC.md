# ToolFlow — UX Спецификация v1.0

> Согласовано в апреле 2026 через сессию тестирования прототипа.
> Этот файл — источник правды для всех UI/UX решений.
> Прототип: ToolFlow_Prototype_v2.html

---

## Навигация

Нижняя панель — 5 кнопок:
- **Все** — все инструменты компании
- **Мои** — только инструменты текущего пользователя
- **Создать** — создание новой карточки (только с правом)
- **Инфо** — статистика и отчёты (только с правом)
- **Панель** — панель управления (только администратор)

Верхние вкладки "Все/Мои" — **убраны**. Навигация только снизу.

---

## Экран входа

- Поля: логин + пароль
- Кнопка "Забыли пароль?" → модалка с запросом к администратору
- Логика: неверный логин/пароль → красное сообщение, не пускает
- Статус "Не работает" → баннер "Доступ запрещён"
- После входа интерфейс зависит от прав пользователя

---

## Карточка инструмента

**В списке (краткая):**
- Фото + категория + статус-чип
- Название (крупно) + инв. номер (моноширинный)
- Владелец: аватар + имя → нажать → контакт с кнопкой "Позвонить"
- Метрики: Владение / Наработка / Обслужен
- Кнопки действий (зависят от статуса и прав)

**Детальная страница (по нажатию на карточку):**
- Большое фото + кнопка "История фото →"
- Поля: категория, инв. номер, серийный номер, статус, владение, последний сервис
- Блок владельца: аватар + имя + должность + кнопка "Позвонить"
  - Если владелец = склад → только название склада, без звонка
- Кнопки: Передать / Сервис / Журнал / Редактировать карточку
  - "Редактировать" — только с правом "Создавать/редактировать карточки"

**История фото:**
- Отдельный экран, сетка 2 колонки
- Каждое фото: кто + дата + тип операции (передача/сервис/создание)

---

## Статусы инструмента

| Статус | Чип | Кнопки |
|--------|-----|--------|
| Активен | зелёный | Передать / Сервис / Журнал |
| В пути (отправитель) | синий пульс | В процессе / Отменить |
| В пути (получатель) | синий пульс | Принять / Отклонить / Журнал |
| На списании | оранжевый | (у склада) Принять списание / Отклонить |
| Утилизирован | тёмный | зачёркнутое название / только История |

---

## Передача инструмента

**Шаги:**
1. Выбор получателя: человек / склад / объект
2. Фото состояния + комментарий (фото необязательно для отправителя)
3. Отправлено → получатель видит уведомление

**Правило фото:**
- Отправитель сделал фото → получатель может принять без фото
- Отправитель НЕ сделал фото → получатель ОБЯЗАН сфотографировать (кнопка заблокирована)
- Хотя бы одно фото на передачу — обязательно

**После отправки:**
- Кнопка "Отменить передачу" — пока получатель не принял

**Отклонение:**
- Инструмент возвращается отправителю + уведомление "Передача отклонена"

**Передача на склад:**
- У ответственного за склад появляется запрос на приёмку
- Ответственный за склад = любой сотрудник с правом "Принимать на склад"

---

## Количественные материалы (BULK)

- Тип BULK показывается в карточке отдельным чипом с единицей измерения
- При передаче: выбор количества (счётчик − / +)
- Показывается: "У [имя]: N единиц · На складе: N единиц"
- Остатки обновляются у обоих после передачи

---

## Сервис и списание

**Типы операций:**
- Плановое ТО
- Ремонт
- Расходники
- Списание (отдельная логика)

**Для ТО/Ремонт/Расходники:**
- Переключатель: "Сделал сам" / "На склад"
- Сделал сам → фото + описание → запись в журнал, инструмент остаётся
- На склад → инструмент уходит на склад, ответственный получает запрос

**Для Списания — 3 шага:**
```
Шаг 1: Сотрудник → запрос с фото + причиной → на склад
Шаг 2: Ответственный за склад → Принять / Отклонить
        При принятии выбирает причину:
        Утеря / Выход из строя / Кража / Расход / Прочее
Шаг 3: Администратор / Директор / Бухгалтер →
        Утвердить утилизацию / Отклонить с комментарием
```

## Уведомления

- Колокольчик с красной точкой в шапке
- Типы: входящая передача / отклонение / запрос на склад / запрос на списание / SMS при добавлении
- Хранятся: всегда
- Каналы доставки: внутри приложения + PWA push (Android) + Telegram-бот

**Telegram-бот:**
- Пользователь привязывает аккаунт один раз в Профиле → @username
- Все уведомления дублируются в Telegram

## Роли и права

**Нет фиксированных ролей** — есть набор галочек у каждого сотрудника.
Администратор выдаёт любые права любому сотруднику.

**Список прав:**
1. Создавать/редактировать карточки
2. Видит только свои предметы
3. Передавать инструменты
4. Передавать без подтверждения
5. Передавать за третьих лиц
6. Принимать на склад / выдавать со склада
7. Подтверждать списание
8. Утилизировать
9. Пополнять количество
10. Создавать отчёты
11. Управлять складами/объектами
12. Управлять пользователями

**По умолчанию** у нового сотрудника все галочки сняты.

**Администратор** — единственная роль которая всегда видит "Панель управления".

## Панель управления

Доступна только администратору. Работает и на телефоне.

**Разделы (боковая навигация):**

### Сотрудники
- Список: ФИО, должность, статус (Работает / Не работает)
- Добавить → форма: Фамилия, Имя, Отчество (необяз.), Телефон +7, Временный пароль → SMS
- Редактировать → ФИО, Отчество (необяз.), Должность, Телефон, Пароль, Статус, Права (галочки)
- При блокировке сотрудника с инструментами → предупреждение, но блокировка возможна

**Должности** (выпадающий список, алфавит):
Администратор, Бухгалтер, директор, Кладовщик, Мастер, Монтажник, Прораб, Разнорабочий, Сварщик

### Склады
- Список складов: название, ответственный, количество позиций
- Создать/редактировать: название + ответственный (из списка сотрудников)

### Объекты
- Аналогично складам
- Объект = стройплощадка, склад = место хранения

### Справочники
Три отдельных редактируемых списка:
- **Категории** (Электроинструмент, Расходники, Оснастка, СИЗ...)
- **Бренды** (Bosch, DeWalt, Makita...)
- **Единицы измерения** (шт, пачка, метр, кг, секция...)

## Экран "Инфо"

Доступен только с правом "Создавать отчёты".

- Метрики: всего инструментов / сотрудников / в сервисе / на списании
- Фильтр по периоду: неделя / месяц / квартал / свой диапазон
- Список ответственных с количеством инструментов
- Кнопка "CSV" → скачать полный экспорт (все инструменты + вся история движений)

## Профиль пользователя

- Фото (загрузка по желанию), по умолчанию аватар с инициалами
- Телефон, логин
- Сменить пароль: текущий → новый → повтор
- Привязка Telegram: @username + инструкция с @ToolFlowBot
